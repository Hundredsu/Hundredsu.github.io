Go命令

编辑器命令

go build：
作⽤：编译 Go 源代码⽂件或包。
示例：go build main.go 会⽣成⼀个可执⾏⽂件 main（在 Windows 上是 main.exe）。

go run：
作⽤：编译并运⾏ Go 源代码⽂件。
示例：go run main.go 会编译并⽴即运⾏ main.go ⽂件。

go fmt：
作⽤：格式化 Go 源代码⽂件，使其符合标准的代码⻛格。
示例：go fmt main.go 会格式化 main.go ⽂件。

go bug：
作⽤：打开 Go 的问题跟踪系统，报告或查看 Go 的问题。
示例：go bug 会打开默认浏览器并导航到 Go 的问题跟踪⻚⾯。

go test：
作⽤：⾃动化测试⼯具，⽤于运⾏ Go 的测试⽂件。
示例：go test 会运⾏当前包中的所有测试⽂件（以 _test.go 结尾的⽂件）。

Env命令
GOOS：
作⽤：指定⽬标操作系统。
示例：GOOS=linux go build 会为 Linux 操作系统构建可执⾏⽂件。

GO111MODULE：
作⽤：控制 Go 模块的使⽤。
值：
off：禁⽤模块⽀持，使⽤ GOPATH 模式。
on：启⽤模块⽀持，忽略 GOPATH。
auto：在项⽬根⽬录有 go.mod ⽂件时启⽤模块⽀持。
示例：GO111MODULE=on go build 会在模块模式下构建项⽬。

GOPROXY：
作⽤：指定 Go 模块代理服务器，⽤于下载依赖。
示例：GOPROXY=https://proxy.golang.org go build 会使⽤指定的代理服务器下载依赖。
GOARCH：
作⽤：指定⽬标体系结构。
示例：go env -w GOARCH=amd64 go build 会为 64 位架构构建可执⾏⽂件。

Module命令

go mod init：
作⽤：初始化⼀个新的模块，创建 go.mod ⽂件。
示例：go mod init example.com/mymodule 会创建⼀个新的 go.mod ⽂件，并将模块路径设置为example.com/mymodule。

go mod download：
作⽤：下载并缓存依赖项。
示例：go mod download 会下载 go.mod ⽂件中列出的所有依赖项。

go mod tidy：
作⽤：添加缺失的模块并移除未使⽤的模块。
示例：go mod tidy 会更新 go.mod 和 go.sum ⽂件，确保它们是最新的。

go mod vendor：
作⽤：将依赖项复制到 vendor ⽬录。
示例：go mod vendor 会将所有依赖项复制到 vendor ⽬录，以便在没有⽹络连接时也能构建项⽬。

go.mod
定义模块的路径及其依赖项。
go.sum
记录模块依赖项的校验和，⽤于验证依赖项的完整性。

启⽤ Go Modules 功能
go env -w GO111MODULE=on

配置 GOPROXY 环境变量，其中-w 表示 写 操作
