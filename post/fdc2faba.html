|浏览量:

我们来展示一下实战攻防的场景（未授权攻击他人服务器是违法行为,请勿模仿)

获取靶机（可以理解为一家公司的四台电脑）中的三份重要文件，可以理解类似商业机密的文件

信息收集

使用fscan，综合漏洞扫描、信息收集的工具进行收集主机的信息，扩展攻击面

扫出thinkphp的rce

使用开发好的漏洞利用工具直接利用

使用一把梭直接梭

连接成功

执行whoami、ifconfig收集内网信息

内网网段为172.22.1.0/24

发现只是普通用户

那就需要提权了

同时这台还是linux

内核容易崩，我们就先考虑suid和sudo提权

我们现在只是普通用户 需要提权到root

mysql的sudo可以提权

https://gtfobins.github.io/

在这个网站查询一下

使用以下的命令即可提权，或者是以root用户来执行命令

攻击成功获取到对面的重要文件即flag01

使用大文件上传fscan，综合扫描工具

还有Stowaway–隧道工具的客户端

这里我直接使用哥斯拉和蚁剑执行fscan进行二次收集一直不成功（这也是在我们真实攻防中经常发生的事，所以我们需要掌握多种攻击手法和多种防御手法）

那我们先建立网络隧道，使得我的电脑可以直接与该服务器的内网进行网络通讯

这里我使用的是Stowaway

其还可以获取shell，同时还可以通过反弹shell使我在我的vps上可以获取交互式shell

vps

先在vps上监听

靶机

在windows和linux上都配置好代理，使得我们的物理机就是我们自己的电脑可以直接和这个服务器的内网的其他电脑进行通讯

反弹shell

由于前面已经给了fscan的执行权限，这里直接执行

对内网进行信息收集

弱口令也可以直接登陆成功

admin

admin123

直接搜索信呼协同办公系统有没有历史漏洞

这里找到一个poc

https://github.com/Threekiii/Vulnerability-Wiki/blob/master/docs-base/docs/oa/%E4%BF%A1%E5%91%BCOA-qcloudCosAction.php-%E4%BB%BB%E6%84%8F%E6%96%87%E4%BB%B6%E4%B8%8A%E4%BC%A0%E6%BC%8F%E6%B4%9E.md

前面挂载好代理了

修改一下exp

一句话木马

因为建立好代理了，所以我们可以直接使用物理机对其内网进行攻击，上传木马

使用蚁剑连接

连接发现直接就是system权限，也就是拿下第二台主机

找到第二个flag

现在就剩域控DC01和172.22.1.21这两台没有拿下了

fscan探测出172.22.1.21有ms17-010，经典的永恒之蓝的漏洞

我们使用kali中的msf直接打

使用代理打开 这样待会的rhost就可以直接填写内网地址了，就不需要再在里面配置代理了

set proxies set proxies socks5:xx.xx.xx.xx:xxxx

稍等一会 攻击成功 shell回来了

现在进行PTH(pass the hash)攻击

使用现成的脚本进行攻击

WMI全称”windows管理规范”，从win2003开始一直存在。它原本的作用是方便管理员对windows主机进行管理。
