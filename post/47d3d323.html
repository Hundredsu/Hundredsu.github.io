先上传fscan进行扫描

然后挂载代理，这里直接vshell挂了

简单整理一下资产

接下来看见邮服了，那肯定先打邮服

F12加ctrl+shift+F进行全局搜索/owa/auth,即可看到当前exchange的版本号。

可以看到是15.1.1591

https://learn.microsoft.com/zh-cn/exchange/new-features/build-numbers-and-release-dates?view=exchserver-2019

那么接下来就是抓取密码，尝试横向其他的机子了

传mimikatz.exe，管理员权限运行

发现抓不到

发现是mimikatz的问题，重新传一个就好了

提取重要的，得到Exchange机器用户哈希和域用户Zhangtong的哈希

那么现在我们就可以通过pth登录Exchange机器用户了

查看XIAORANG-EXC01$是否在以下三个组内

Microsoft Exchange Security Groups

Exchange Trusted Subsystem

Exchange Windows Permission

上面的信息收集可以用bloodhound进行收集（但是实战用不了）

exchange机器账户隶属于Exchange Windows Permissions这个组中，而这个组具有write 域内acl权限，所以可以设置刚刚通过mimikatz拿到的域内用户Zhangtong的dcsync属性。

这里注意域名和用户名

直接给Zhangtong添加DCSync权限

在域环境中，如果安装Exchange后，系统会添加一个名为Microsoft Exchange Security Groups、Exchange Trusted Subsystem和Exchange Windows Permission三个组。如果获得了这三个组内任意用户的控制权限，就能够继承用户组的WriteDACL权限，WriteDACL权限可以修改域对象的ACL，最终实现利用DCSync导出域内所有用户hash。

修改成功

进行DCSync攻击

抓取到administrator的hash

进行pth获取域控权限

成功
